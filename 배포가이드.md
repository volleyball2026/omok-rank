# 🏆 우리 반 오목 랭킹 — 배포 가이드 (GitHub + Vercel + Supabase)

- **DB + 백엔드 로직**: Supabase (Postgres + SQL 함수)
- **프론트엔드(정적 HTML)**: Vercel
- 별도의 파이썬/서버가 필요 없습니다. 프론트가 `supabase-js`로 DB를 직접 호출합니다.

파일 구성
```
index.html          화면 + 게임 전체 (supabase-js 사용)
omok.js             오목 렌주룰 규칙 엔진 (순수 로직)
omok_coach.js       오목 코치 엔진 — 모양·이중위협·VCF 수읽기·정석 판별 (복기/배우기가 사용)
omok_ai.js          오목 AI 엔진 — 코치 엔진을 써서 다음 수를 고른다 (AI 연습 대국이 사용)
chess.js            체스 규칙 엔진
janggi.js           장기 규칙 엔진
supabase_schema.sql Supabase에 붙여넣을 테이블/함수
*_test.mjs          엔진 테스트 (개발용, 배포엔 불필요)
```
> 배포에는 `index.html`, `omok.js`, `omok_coach.js`, `omok_ai.js`, `chess.js`, `janggi.js` 6개 파일이 필요합니다.

> ### ⚠️ 이미 운영 중인 분께 — 이번 업데이트는 DB 작업이 필요합니다
> **최근 경기 피드**가 랭킹전·친선전·온라인을 모두 보여주게 바뀌면서 `game_records` 테이블에
> 컬럼 3개(`ranked`, `online`, `lp_change`)가 추가됐습니다.
> Supabase → **SQL Editor** 에 `supabase_schema.sql` **전체를 다시 붙여넣고 [Run]** 해주세요.
> 모든 구문이 `if not exists` 라서 **여러 번 실행해도 안전하고, 기존 데이터도 그대로 유지**됩니다.
> (이 작업 전까지는 새 경기가 피드에 안 쌓입니다. 기존 경기 기록은 `친선전`으로 표시됩니다.)

## 1단계 · Supabase 프로젝트 만들기 (DB)

1. https://supabase.com 접속 → 로그인 → **New project**
2. 프로젝트 이름/비밀번호(DB password) 입력 → Region은 **Northeast Asia (Seoul)** 권장 → 생성(1~2분)
3. 왼쪽 메뉴 **SQL Editor** → **New query** →
   `supabase_schema.sql` 파일 내용을 **전체 복사해 붙여넣고 [Run]**
   → `Success` 가 뜨면 테이블(`students`, `match_history`)과 함수(`record_match`)가 만들어집니다.
4. 왼쪽 **Project Settings(톱니바퀴) → API** 로 이동해 아래 2개를 복사해 둡니다:
   - **Project URL** (예: `https://abcd1234.supabase.co`)
   - **anon public** 키 (`eyJ...` 로 시작하는 긴 문자열)

> anon 키는 브라우저에 노출돼도 되는 공개 키입니다.
> 스키마의 RLS 정책이 조회·학생등록만 허용하고, 점수 변경은 `record_match` 함수로만 가능하도록 막아둡니다.

## 4단계 · Vercel로 배포 (프론트엔드)

1. https://vercel.com 접속 → GitHub 계정으로 로그인
2. **Add New… → Project** → 방금 올린 `omok-rank` 저장소 **Import**
3. 설정은 **기본값 그대로** 둡니다 (정적 사이트라 빌드 설정 불필요):
   - Framework Preset: **Other**
   - Build Command: (비움)
   - Output Directory: (비움 — 루트의 `index.html`을 그대로 서빙)
4. **Deploy** 클릭 → 잠시 뒤 `https://omok-rank.vercel.app` 같은 주소 발급 🎉

이 주소를 학생들에게 알려주면 각자 기기로 접속해서 **랭킹 확인 + 오목 대국**을 할 수 있습니다.

> 이후 코드를 고쳐 `git push` 하면 Vercel이 자동으로 다시 배포합니다.

규칙·코치·AI 엔진을 고쳤다면 배포 전에 엔진 테스트를 돌려 보세요(DB 없이 동작합니다).

## 사용 방법 (교실)

1. **랭킹 탭**에서 `+ 학생 등록`으로 반 학생들을 등록합니다.
2. **오목 두기 탭**에서 흑돌/백돌 학생을 고르고 `대국 시작!` →
   한 기기(태블릿/노트북)에서 번갈아 돌을 놓습니다.
3. 5목이 완성되면 **자동으로 승패가 랭킹에 반영**되고 티어/LP가 갱신됩니다.
4. 실물 오목판으로 둔 경기는 `⚔️ 결과 직접 입력`으로 승자/패자만 선택해 기록할 수 있습니다.

## 주요 기능

- **🔐 계정(비밀번호)**: 등록 시 이름+비밀번호를 정합니다. 오목 대국을 시작하려면 **흑·백 두 학생 모두 자기 비밀번호를 입력**해야 하고, 실물 경기 `결과 직접 입력`도 양쪽 비밀번호를 확인합니다. (남의 이름으로 대국·기록하는 실수 방지)
- **🔧 관리자 모드**: 랭킹 탭 맨 아래 `🔒 관리자 모드` → 관리자 비밀번호 로그인. 학생별 **포인트·LP·티어·승/패/연승 조정**, 이름 변경·삭제·비밀번호 재설정, 시즌 초기화, 관리자 비밀번호 변경. **관리자 비밀번호는 `2875`** (관리자 모드에서 변경 가능).
- **🧪 테스터 계정**: 관리자 모드에서 학생을 `테스터`로 지정하면 **랭킹에서 빠지고, 그 학생이 낀 경기는 랭킹전을 골라도 자동으로 친선전이 됩니다**(LP·티어 변동 없음). 시험 삼아 대국해 볼 때 랭킹이 더러워지지 않게 하는 장치예요.
  - 대국 설정에서 테스터를 고르면 **시작 전에 안내가 뜹니다**. 온라인 대국도 마찬가지로 친선전 처리되고, 최근 경기에도 `🤝 친선전`으로 남습니다.
  - 즉 **테스터와 둔 경기가 친선전으로 보이는 건 정상**입니다. 랭킹에 반영하고 싶으면 관리자 모드에서 `테스터` 지정을 해제하세요.
- **📱 가로모드 최적화(태블릿)**: 대국 화면을 가로로 돌리면 판이 화면 높이에 맞춰 크게 커지고, 플레이어·옵션·버튼이 판 오른쪽에 세로로 배치됩니다. 세로일 때는 "가로로 돌리세요" 안내가 표시됩니다.
- **🔼 승급까지 LP**: 랭킹 카드마다 다음 티어까지 남은 LP를 표시합니다.
- **🛍️ 포인트 상점 (스킨)**: 경기하면 포인트가 쌓이고(승 +20P·패 +8P), **상점 탭**에서 바둑판/바둑알 스킨을 사서 장착합니다. 장착한 스킨은 오목 대국에 실제로 반영됩니다 — 각 플레이어의 바둑알 스킨이 자기 돌에, 흑 플레이어의 판 스킨이 판에 적용됩니다. (롤 스킨처럼! 가격은 서버가 관리해 조작 불가)
  - 바둑판 6종(기본/한밤/벚꽃/숲속/바다/황금), 바둑알 7종(기본/옥/루비/하늘/황금/흑요석/은하). 스킨 가격·종류는 `supabase_schema.sql`의 `shop_items` 목록에서 자유롭게 추가·수정할 수 있습니다.
- **🎉 승리 컨페티**: 5목 완성 시 색종이 애니메이션.
- **실시간 동기화(Realtime)**: 한 기기에서 결과가 기록되면 접속 중인 모든 기기의 랭킹이 자동 갱신됩니다. (교사 화면을 TV에 띄워두면 실시간 순위판이 됩니다)
- **연승 표시 🔥**: 2연승부터 이름 옆에 연승 수가 표시됩니다.
- **📜 최근 경기 피드**: 랭킹 탭 아래에 최근 10경기가 나열됩니다. 각 줄에 **종목(⚫오목/♟️체스/🏯장기)·🏆랭킹전 또는 🤝친선전·🌐온라인 여부·시각**이 표시되고, 랭킹전은 승자의 획득 LP도 함께 나옵니다. (오늘 경기는 `14:32`, 이전 경기는 `7/16 09:10` 형식)
- **📅 더보기 — 날짜별 전체 기록**: 피드 아래 `더보기`를 누르면 최근 500경기를 **날짜별(오늘·어제·날짜)로 묶어서** 볼 수 있습니다. **종목별**(전체/오목/체스/장기), **랭킹전·친선전**으로 걸러볼 수 있어요.
  - 랭킹전·친선전·온라인 대국이 **모두** 기록됩니다(예전에는 랭킹전만 피드에 나왔습니다).
- **금수 표시(오목)**: 게임 화면의 `금수 표시`를 켜면 흑 차례에 둘 수 없는 자리(삼삼·사사·장목)가 빨간 ✕로 표시됩니다. 아이들이 렌주룰을 배우기 좋습니다.
- **📖 복기 — AI 코칭**: 랭킹에서 학생을 눌러 `복기`를 열면 최근 대국을 한 수씩 되짚어 볼 수 있습니다. 오목은 **수마다 등급(⭐최고의 수 / 👍좋은 수 / ❗실수 / 💥큰 실수)과 이유**가 붙습니다.
  - 실전 개념으로 평가합니다: **열린3·열린4·사삼(4-3)·삼삼(3-3)** 같은 이중위협을 만들었는지, 상대 위협을 놓쳤는지, **VCF(4로 계속 몰아 이기는 수순)** 를 놓쳤는지까지 읽어 줍니다.
  - **더 좋았던 자리는 판 위에 초록 점선**으로 직접 표시됩니다. 위쪽에는 대국 리포트(흑/백 각각 최고의 수·실수 개수)가 나오고, `❗ 아쉬웠던 수로 이동` 버튼으로 실수만 골라 볼 수 있습니다.
  - 첫 3수가 정석이면 **렌주 26주형 이름(화월·장성 등)** 을 알려줍니다.
- **🤖 AI 연습 대국 (오목)**: 오목 탭 맨 위 `AI와 연습 대국`. **혼자서** 연습할 수 있어 짝이 없어도, 집에서도 둘 수 있습니다.
  - 난이도 3단계: **🐣 초급**(이제 막 배운 친구 — 5목 막기를 가끔 깜빡합니다) / **🐤 중급**(사삼·삼삼 이중위협을 만들고 막습니다) / **🦅 고급**(VCF까지 수읽기합니다).
  - **내 돌(흑/백)을 고를 수 있습니다.** 백을 고르면 AI가 먼저 둡니다. 상점 로그인이 되어 있으면 **내 이름과 스킨**으로 둡니다.
  - AI가 왜 그 수를 뒀는지 **말풍선**으로 알려 줍니다(“열린3! 이거 막아야 해 😏”, “4로 계속 몰아서 이긴다 ⚡”). 배우면서 두라고 넣은 기능입니다.
  - 판이 끝나면 `📖 이 판 복기하기`로 **바로 복기**할 수 있습니다(위의 AI 코칭과 같은 화면).
  - **랭킹·포인트·경기기록에 전혀 반영되지 않습니다**(연습으로 점수를 벌 수 없게). 대신 **레벨별 전적(3승 1패 …)이 기기에 저장**되어 목표가 생깁니다.
  - `무르기`를 누르면 **내 수와 AI의 응수가 함께** 취소돼 다시 내 차례가 됩니다. 연습이니 마음껏 물러도 됩니다.
- **🎓 오목 배우기 — 11단계 문제**: 마이페이지의 `오목 배우기`. **기본기 → 모양 익히기 → 이중위협 → 초반 정석** 순서로 실력이 올라가게 짜여 있습니다.
  - 기본기: 5목 완성 / 상대 5목 막기 / 사이를 메워 5목
  - 모양: 열린3 만들기 / 열린4 만들기 / 흑의 금수 찾기
  - 이중위협(핵심): 사삼(4-3)으로 이기기 / 사삼 마무리하기 / 삼삼(3-3)으로 이기기 / 상대의 사삼 막기
  - 정석: 화월 두기
  - 정답은 엔진이 직접 판정해서 **대칭·별해도 모두 정답 처리**되고, 틀리면 **왜 틀렸는지** 알려 줍니다. `💡 정답 보기`로 정답 자리를 볼 수 있습니다. 진도는 기기에 저장됩니다.
- **소리/진동**: 돌 놓기·승리·금수 시 효과음과 진동. `소리` 토글로 끌 수 있습니다.
- **재대국**: 게임이 끝나면 `🔄 색 바꿔 다시`로 같은 상대와 흑백을 바꿔 바로 한 판 더.
- **무승부**: 판이 가득 차면 무승부 처리(점수 변동 없음).
- **학생 관리**: 학생을 눌러 상세창에서 `이름 수정`·`삭제` 가능.
- **시즌 초기화**: 랭킹 탭 맨 아래 `시즌 초기화`로 티어·LP·전적을 리셋(2번 확인). 학기/시즌이 바뀔 때 사용하세요. **포인트와 구매한 스킨은 유지됩니다**(아이들이 산 스킨을 잃지 않도록).

## 규칙 요약

**티어 / 점수(LP)**
- 티어: 브론즈 → 실버 → 골드 → 플래티넘 → 다이아몬드
- 승급: 100LP 도달 시 다음 티어 (초과분 이월). 다이아몬드는 최고.
- 강등: 0LP에서 더 지면 아래 티어로. 브론즈 밑으로는 강등 없음.
- Elo 가중치: 동등 ±15, 티어 차이 1당 ±5 (하한 10 / 상한 30).
  하위가 상위를 이기면 최대 +30, 상위가 하위를 이기면 최소 +10.

**오목 (정식 렌주룰)**
- 15×15 판, 흑 선공.
- 흑 금수(둘 수 없음): **삼삼(3-3), 사사(4-4), 장목(6목 이상)**.
- 흑은 정확히 5목으로만 승리. 백은 5목 이상이면 승리(제약 없음).
- 5목 완성은 금수보다 우선하여 승리로 인정.

> 참고: 대회 수준의 재귀적 금수 판정(금수점을 이용한 3·4의 성립 여부)까지는 다루지 않지만,
> 교실 대국에서 실제로 나오는 삼삼·사사·장목은 정확히 판정합니다.
> (검증 테스트: `node omok_test.mjs` / `omok_coach_test.mjs` / `lessons_test.mjs` / `chess_test.mjs` / `janggi_test.mjs`
> — 현재 146개 케이스 전부 통과. 코치 엔진의 26주형 좌표는 국제렌주연맹 공식 다이어그램 기준입니다.)

## ⚠️ 데이터 보존 참고

Supabase 무료 플랜은 **데이터가 영구 저장**되므로 Render 무료 플랜처럼 초기화되지 않습니다.
단, 무료 프로젝트가 **7일 이상 아무 접속이 없으면 일시정지(pause)** 될 수 있는데,
대시보드에서 **Restore** 하면 데이터 그대로 다시 살아납니다. (수업 중엔 매일 접속하므로 문제 없음)
